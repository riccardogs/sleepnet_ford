2. models/simple_sleep_net.py
--------------------------------------------------------------------------------
MODIFICA: Kernel_size, stride, padding per tutti e 3 i blocchi convolutivi
RIGA: ~70-130

PRIMA (EEG - 3000 campioni):
   Conv1: in=1, out=32, kernel=64, stride=8, pad=63, dil=1
   Conv2: in=32, out=64, kernel=32, stride=4, pad=62, dil=2
   Conv3: in=64, out=128, kernel=16, stride=2, pad=60, dil=4
   Linear: 128 → latent_dim (128)

DOPO (FordA - 500 campioni):
   Conv1: in=1, out=16, kernel=16, stride=4, pad=7, dil=1
   Conv2: in=16, out=32, kernel=8, stride=2, pad=5, dil=2
   Conv3: in=32, out=64, kernel=4, stride=2, pad=4, dil=4
   Linear: 64 → latent_dim (64)

PERCHÉ: 
- Kernel troppo grandi (64 su 500 campioni vs 3000 originali)
- Padding eccessivo (63 su 500 è ~12% del segnale)
- Stride ridotti per preservare più informazioni temporali
- Riduzione filtri da 128 a 64 per evitare overfitting (dataset più piccolo)
- latent_dim ridotto da 128 a 64 per rappresentazione più compatta

================================================================================
COSA CAMBIA TRA FORDA E FORDB (NON LA STRUTTURA, MA IL CONTENUTO)
================================================================================

1. CONDIZIONI DI ACQUISIZIONE
   - FordA: Condizioni operative standard (temperatura normale, carico normale)
   - FordB: Condizioni operative variabili (diverse temperature, regimi motore)

2. DIFFICOLTÀ DEL TASK
   - FordA: Più facile (pattern anomali più distinguibili)
   - FordB: Più difficile (test di generalizzazione/robustezza)

3. PERFORMANCE ATTESE
   - FordA: Accuracy più alta (~87% con 1 epoca, ~92-95% con training completo)
   - FordB: Accuracy più bassa (~75-80% con 1 epoca)

4. CONTESTO DI UTILIZZO
   - FordA: Benchmark principale per anomaly detection su motori
   - FordB: Test di robustezza e generalizzazione

================================================================================
PERCHÉ IL DATALOADER È GENERICO
================================================================================

Il dataloader (forda_loader.py) è stato scritto per essere GENERICO:

    data = np.load(self.data_path)
    X = data['X']
    y = data['y'].astype(int)
    y = (y + 1) // 2
    if len(X.shape) == 2:
        X = X.reshape(X.shape[0], 1, X.shape[1])

NON contiene:
   - Nomi di file specifici (es. "SC4001.npz")
   - Numero fisso di campioni (es. 4921)
   - Path hardcodati (li legge dal config)

ACCETTA QUALSIASI FILE .npz che abbia:
   - Chiave 'X' con shape (N, 1, 500) o (N, 500)
   - Chiave 'y' con classi -1/1 o stringhe '-1'/'1'

================================================================================
PERCHÉ IL MODELLO È GENERICO
================================================================================

Il modello (simple_sleep_net.py) è stato modificato per 500 campioni:

    self.conv_path = nn.Sequential(
        nn.Conv1d(1, 16, kernel_size=16, stride=4, padding=7),  # 500 → 125
        nn.Conv1d(16, 32, kernel_size=8, stride=2, padding=5),  # 125 → 62
        nn.Conv1d(32, 64, kernel_size=4, stride=2, padding=4),  # 62 → 31
        ...
    )

NON DIPENDE DA:
   - Numero specifico di campioni (funziona con QUALSIASI N)
   - Nome del dataset
   - Numero di classi (2 classi, ma potrebbe funzionare anche con altre)
